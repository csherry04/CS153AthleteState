# Coaching Agent

The coaching agent is a local, tool-grounded interface for asking questions about the athlete monitoring data. It answers from generated CSV/JSON outputs rather than generic training advice.

## Current Flow

There are two user-facing coach surfaces:

1. **Coach live** (`web/src/pages/Coach.tsx`)
   - Calls the local FastAPI backend in `src/coach_api.py`.
   - Uses `OPENAI_API_KEY` from the environment or local `.env`.
   - Grounds answers through `src/agent_tools.py`.

2. **Coaching QA static** (`canvases/coaching-qa.canvas.tsx`)
   - Shows curated example questions and answers.
   - Generated by `scripts/generate_coaching_qa.py`.
   - Does not call an API.

The standalone CLI agent in `scripts/run_coaching_agent.py` is still useful for local testing and demo mode.

## Quick Start

Run the offline demo:

```bash
python scripts/run_coaching_agent.py --demo
```

Run the live web coach:

```bash
export OPENAI_API_KEY=your_key_here
python scripts/run_coach_api.py
cd web
npm run dev
```

Then open the web app and use `Coach (live)`.

## Tools

`src/agent_tools.py` exposes grounded tools for:

- retrieving a day’s monitoring scores and recommendation
- comparing days or periods
- explaining model disagreement
- analyzing progression, recovery context, and workouts
- simulating simple volume/down-week adjustments
- reading frontier outcome evaluation and athlete profile summaries

The tools read from generated outputs such as:

- `outputs/analysis/athlete_bone_stress_scores.csv`
- `outputs/analysis/date_explorer.json`
- `outputs/analysis/frontier_outcome_evaluation.json`
- `outputs/analysis/athlete_bone_stress_periods.csv`
- `outputs/analysis/athlete_profile.json`

## Static Examples

Regenerate the static examples page with:

```bash
python scripts/generate_coaching_qa.py
```

This writes:

- `outputs/analysis/coaching_qa.json`
- `canvases/coaching-qa.canvas.tsx`

The static page is intentionally illustrative. It should show what good coach answers look like without depending on an API call.

## Live Backend

The live backend route is:

```text
POST /api/coach
```

The backend does three things:

1. Uses local fast answers for common questions when possible.
2. Prefetches obvious tool results for questions about dates, risk, trends, progression, or recovery.
3. Falls back to the chat completions API with tool definitions when synthesis is needed.

Responses are cleaned before display so hidden tool syntax or malformed tool markup does not appear in the UI.

## Validation

Useful checks:

```bash
python -m py_compile src/coach_api.py src/agent_tools.py scripts/run_coach_api.py scripts/run_coaching_agent.py
python scripts/run_coaching_agent.py --demo
cd web && npm run build
```

## Caveats

- The live coach reads the latest generated analysis outputs. Regenerate scores/pages when the underlying data changes.
- Counterfactuals are heuristic unless a specific simulator reruns the relevant scoring path.
- This is a monitoring and explanation interface, not a medical diagnosis tool.
